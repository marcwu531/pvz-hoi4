Useful Info

  • Adobe Animate: Use this program to animate everything in PvZ besides the particles. Follow Raducu’s guide and video about animation.
  • Paint.NET/Photoshop: These are the most popular programs that can be used to draw assets for the game.
  • XML: The file extension for all particle effects, this can be edited with Notepad but has to be compiled and decompiled with PopStudio.
  • FLA: The file extension that all source PvZ animations use. You must open them with Adobe Animate
  • FlashReanimExport.jsfl: This is a script you need to use to export all your flas into reanims. Drag it into animate and it will export a .reanim with the file name into the folder with the fla. This may take a considerable amount of time depending on your PC specs and the general fla size itself.
  • Reanim: This is the format all PvZ animations turn into after using the script. It can still be edited but only through notepad. It’s also easy to change file names while in this format instead of re-exporting it.
  • Compiled: This is the compressed version of a reanim. You cannot edit this whatsoever but can be decompiled by the programs below.
  • PvZUtil: Used for compiling reanim files. unlock icon decompiles .compiled files, lock icon compiles reanims. do not use this for xml's.
  • PopStudio/PvZStudio: PvZUtil but extra functionality, can unpack multiple versions of the game, create atlases, encode and decode files, and compile and decompile files. It’s highly recommended to learn how this works.

The Ultimate Asset Pack

Includes every frame from every fla file on the PC version. If you can’t find what you want in this pack, here’s how to make your own.

  1. Open whatever FLA you want, and go to the library with all the assets.
  2. To make it easier on yourself scroll to the side of the assets to find the file type and sort by bitmap. You need to double click on all the assets to make sure they are all (Lossless PNG/GIF). This is so the assets don’t have jpeg compression when they’re exported.
  3. Select every frame in the fla and copy them.
  4. Create a new symbol and make sure the type is Graphic.
  5. Paste all the frames inside the graphic.
  6. Right click on the graphic and click Export as PNG Sequence, you can name this whatever you want.
  7. Make sure the Colors is at 32 bit and you checkmark Smooth, you can also change the Resolution to have it exported at a higher quality, the lower the resolution the more pixelated it will be. I used 350 dpi for all the exports in the UAP.
  8. Extra step, if you want to add them into a sprite sheet you can use https://www.codeandweb.com/free-sprite-sheet-packer.

(DISCLAIMER) Some may be irritated that the size of these frames are so large, the reason for that being that when exported at a small scale, the frames become incredibly pixelated, making them look terrible. If you want to make the frames size accurate to the in-game ones, scale them down by 20.5%. Crazy Dave needs scaled down by 36%

Q&A of the Basics

Q: What do I need to start animating?

A:  Adobe Animate, PvZUtil/Popstudio, and the FLAs for every plant and zombie.

Q: How do I export my animations?

A: The script named FlashReanimExport.jsfl needs to be dragged into Adobe Animate. A .reanim file will be inside the folder where the .fla is. Be patient as it takes a while to export everything.

Q: How do I put my animations in the game?

A: Compile the reanim with PvZUtil or Popstudio, read their associated guides on how to use them, then copy and paste the compiled animation in the compiled>reanim folder.

Q: My animation is crashing the game. How do I fix this?

A: There are many reasons why this could be happening. If the FPS is higher than 12 then you need to lower it in Adobe Animate or compile with Popstudio. The files for the animation might not be inside the folder or named incorrectly so make sure every file is inside the reanim folder, tween names do not matter. If these issues still persist after fixing these then request help in the PvZ MA Server.

Adobe Animate Guide

Adobe Animate is one of the most important parts of modding PvZ visually. It might seem intimidating at first but after reading this guide you should be able to understand most of the basic features of it.

Let's use Blover.fla as an example.

There is a lot on the screen right now, but all will be explained right here.

The main window at the top of the screen is called the timeline as it has everything useful pertaining to animating.

First, there are layers. Every asset you see in this image has a layer to it. The head, stems, petals, and dirt are all separated so they can be animated.

Then, there are the dark gray and gray rectangles you see. Those define whether or not an image is inside the layer on that frame. Dark gray means that nothing is on that frame while gray means that 1 or more assets are in the frame. For PvZ animations you cannot have more than one asset in the frame without it not functioning in-game.

The gray rectangles can be turned into something called Classic Tweens. This allows the asset to move around freely instead of moving the asset frame by frame. Most of, if not, every animation in flash or animate uses tweens, and they aren’t hard to use either. It’s all trial and error to see what works and what doesn’t. Studying the animations of the game can benefit you in making animations that are accurate to the game's style.

The black dots you see are Keyframes. These keep track of the position of an asset throughout the timeline. This is essential to making animations and would be impossible without them. You can make something a keyframe by right clicking and clicking “Insert a keyframe” or by pressing F6 which converts a frame into a keyframe. The key to making these work is by having some space between each keyframe and making sure each of them is part of the major motion and letting the tweening do the rest. If something looks off in-between the motion then you can add another keyframe to said part to adjust and fix the issue. Don’t forget to not add too many frames because PvZ has interpolation which makes all the animations smooth so don’t pay too much attention to the low fps count. You also want to make sure the animation has gravity and some physics to it, almost everything in the game has a small jerking animation after a bounce. For example, the Blover has its petals jerk slightly downwards and upwards after each major motion, being the head bounce. As stated above, studying the game's animation is incredibly useful for recreating the style.

Now for the White dot that’s present on all assets.

This is called the anchor. Its purpose is to change the center of where something is resized or rotated. You need to make sure you don’t change where this is located after a keyframe as the asset will usually fly off or do a completely random and unwanted movement. It’s good practice to put the anchor in the area you want it to be before you start making the animation so you don’t have to worry about it later on.

This is the toolbar, it contains all the tools and more for animating

The first is the Selection Tool. This allows you to click and drag assets to any position on the screen. Dragging on an open area allows you to make a selection square so you grab the assets you want in an area. Holding Shift+Clicking allows you to select multiple assets.

The second is the Transform Tool. This allows you to transform your assets into different sizes. Holding Shift while dragging on one of the corners allows you to keep its original size. This is one of the best tools to offer so make good use of it.

The third is Lasso Tool. This is a freeform selection square mentioned above.

This next section includes many art related tools. It should be noted that the game will crash if you use any of these. The one that will be mentioned is the Line Tool. This is used in many of the guide layers seen in other FLAs to keep track of a certain position something needs to be in. The reason the game doesn’t crash is because those layers have an underscore in front of the name. When using the FlashReanimExport.jsfl it skips layers with an underscore in the front so keep this in mind to not export unwanted layers. The rest of the tools in this section are brushes, erasers, and a rectangle tool all for the purpose of drawing.

The next section includes mostly the same things regarding art. A pen too, a text tool, a paint fill tool, an eyedropper tool, and a warping tool. All of these relate to the last set of tools so they won’t be much help to us here besides the text tool.

The last set of tools are the rotation tool and the zooming tool. The Rotation Tool allows you to rotate the canvas. You can reset the rotation by clicking the Center Stage button. The Zoom Tool allows you to zoom in or out depending on the setting you have it as. You can also do this by holding Ctrl+Scroll Wheel in any direction.

This set of tools is located at the top of the stage window.

The first tool is the Center Stage button. This allows you to put the canvas back into the center after you have moved it around.

The second tool is the Rotation Tool. This rotates the entire canvas.

The third tool is the Stage Mask Tool. This allows you to crop the canvas to whatever dimension it is at. This isn’t too useful for use since the game only uses the canvas to position the animation.

The last tool is the Zoom Percentage. This comes with zoom presets if you want to select a manual zoom level instead using Ctrl+Scroll Wheel. The arrows control the zoom by 1.

Next are these four icons located at the top left of the timeline right where the layer names are.

The first one is a layer highlighter. It underlines the layer with a color of your choosing. This can be used to keep track of editing layers with priority.

The second one is a layer outliner. This will outline the asset in the layer with a color of your choosing while also making the asset itself invisible.

The third one is to show or hide the layer. It's pretty self explanatory and can be beneficial when you want to focus on certain details of the animation without having to label it.

The fourth and final one is the lock. This makes it so you cannot edit anything on the layer besides adding and deleting frames.

These are the icons located towards the center of the timeline.

The first set of icons goes in the order, insert keyframe, insert blank keyframe, insert frame, auto insert keyframe, and remove frame

The icon with two circles is called Onion Skin. This allows you to view a range of frames with two colors of your choosing. This is if you need a reference for the future and previous frames while also viewing the current frame.

The three blanked out icons are insert classic tween, insert motion tween, and insert shape tween.

The final two are the repeat button and play button. The repeat button can have its range changed to only view certain sections of your choosing.

The next topic is the assets used in the animation.

All animations need assets, and there's a lot that goes into using those assets. Usually you start your projects by dragging all of your assets into the library. Next, you should properly size each asset onto their own layer and make sure it aligns to what you are trying to accurately animate, using a guide image can make this process much easier, as well as having a plant/zombie reference to make sure you’re sizing it correctly. Then, you want to select each layer and convert them all into classic tweens. This converts the assets used into Symbols which is represented by a gear icon next to the name

Symbols are practically mandatory for animating. They allow you to make easy texture swaps if you need to update an asset or to resize them and whatnot. They have many benefits but also have some restrictions. You may notice that you can create animations and add layers inside the symbol. These things won’t properly translate to the in-game engine and will not be visible at all. While you can’t add layers, you can add multiple images to a single layer and it will show up in game. Blover does this with its petals to save the effort of having to animate three separate petals

You may have noticed the gray rectangles at the bottom of the timeline with names such as anim_idle and anim_shooting. These are necessary for the game to recognize what parts of the animation are what. This is done by adding a blank symbol called locator on the layer and naming it to whatever the action is and visible until said action has ended. You can see in the first image in the anim_idle layer that the gray rectangle ranges for a certain amount of frames which will determine how much of the animation is played. Adding these yourself can be difficult because you need to have the exact name or else it will not play correctly. This is why building off of plants is easier since a lot of the ground work is already there for you.

If you want to build an animation from the ground up, there are a few things to keep in mind. One of these things is the canvas size. You want to make sure that if you’re making a plant that it’s 80 x 80, zombies are usually 80 x 139. This can be made easier if you know what plant you are replacing. Next is the frame rate, which should always be 12 fps. You can determine this by looking at the top of the timeline. Most of the time it is, but when making your own animation you should stick with 12 so you don’t run into any problems.

Now it's time for the properties of keyframes

This is the Frames tab, it allows you to make many edits to the keyframes.

The first section is Label. This allows you to add a label to the keyframe with 3 types. These are name, comment, and anchor.

The second section is Tweening. These settings can affect the movement of the tweens. The first setting, Ease, allows you to either add effects all together or separately. The second setting, Effect, allows you to give an easing effect to tween. These are the premade effects given.

No Ease: applies no effect to the animation

Classic Ease: allows you to adjust the intensity with a slide going from -100 to 100

Ease In: eases into a movement, this comes with 9 premade modifiers which change the intensity of the ease.

Ease out: eases out of a movement, comes with the same premade modifiers

Ease In Out: eases into and out of a movement.

Custom: allows you to make your own ease modifiers. A curve at the bottom signifies easing in while a curve towards the top signifies easing out.

I recommend not using this as almost none of the PvZ animations use easing, it doesn’t match the style of the game. If you want to make a good looking animation akin to PvZ2 or just a nice looking animation in general then this is a great feature to have.

The third and final setting is Rotate. This applies a clockwise or counterclockwise rotation to the tween. You can adjust the intensity with the slider to the right of it.

The final section is Sound. This isn’t particularly useful for us since we aren’t making anything that needs sound in these animations.

The next tab is the Object tab. This controls the positions and effects of keyframes.

The Position and Size section allows you to precisely move and resize keyframes. You can also move them with your arrow keys too.

The Color Effects section allows you to add effects to the keyframe. There are multiple ones available, but the only one able to be viewable in-game is the Alpha effect. This controls the visibility of the layer and comes with a slider to control that. Scaredy-Shroom, Cherry Bomb, and Doom-Shroom are one of the more noteworthy examples of this effect being used.

The final tab is the Doc tab. This controls everything relating to the file itself

You can change the size, stage color, and FPS of the doc. The match contents button makes the stage size match how far the assets reach out.

Applying it to the pasteboard makes the whole bottom screen match the color of the stage.

*Things to Note*

  • Most of, if not all, of the pea shooting plants have their anim_shooting frame cut off 13 frames into the animation after it plays for the first time.
  • On the majority of the zombie fla’s the outerarm_hand, outerarm_lower, anim_head1, anim_head2, anim_hair layers all need to have the same name for their limbs to disappear correctly. outerarm_upper needs to have the same name so it switches to the bone. You can name other layers with these names to make them disappear as well.
  • anim_stem on the pea shooting plants controls where their head is positioned.
  • The Boss_head2 layer in the zombie_boss.fla controls where zomboss will be.
  • Fla's with a frame rate higher than 12 such as the majority of the Zen Garden Animations and the Digger Zombie need to be compiled with popstudio to work correctly, if lowered manually they will play slower in game, if compiled with pvzutil they will not work. (this is an exception for some anims in games but it's wiser to use popstudio)
  • Fla’s that use alpha’s such as Cob Cannon and Scaredy-Shroom need to be compiled with popstudio to work correctly, if not the layers that use it will vanish in game.
  • If you want to make a custom sun production animation, note that it takes 30 frames in game for the sun to come out, unless using scripts.
  • The main menu can be edited. Although the placement of the zen garden, almanac, shop, options, help, and quit are all predetermined by the tombstone layer. The trophy is controlled by the tree layer. You can use a template for making menus line up in-game here. You can also replace the images in SelectorScreen.fla to make them line up as well.
  • If you want to recreate the mobile zombies with the bigger head, multiply the width and height of every head symbol by 1.3 in adobe animate.
  • The GOTY versions of the dancing zombie and its backup dancers are 1.252% bigger than every other zombie. You can use this percent to fix their height in other versions or scale up zombies to replace it in the GOTY version.

How to Recreate the PvZ Art style

A great place to get started when trying to recreate this art style is by using reference material. Rich Werner has his own website that includes lots of his work relating to the making of PvZ, it can be viewed Here. You can use his site to also view how he approaches certain designs and what to keep in mind.

The next thing you need is an art program. In the Useful Info section, it mentions Paint.net and Photoshop. There are plenty of other art programs out there such as Gimp and PaintToolSai which can give accurate results if you are skilled enough. PvZ was drawn in Photoshop so if you want to be 100% accurate then you will have to obtain it legally (lol). Unfortunately, the brushes he used were custom-made, so it's up to you to recreate the brushes.

Paint.net is a simple art program that can be used to make quick edits due to its quick and simple interface, this does also mean that its tools are more limited than programs like Gimp and PaintToolSai due to them being more focused towards art.

One of the main things you will see in the PvZ Style is the somewhat painting style they all have. Rich makes paintings so his skills and art style transfer into the game’s art style, you can notice the individual brush strokes on some of the plants and zombies. You want to make this somewhat apparent and not too much of a straight gradient from one color to another, something more patchy in a way.

Plants are generally smoother and have light strokes across them. It may also look like there are a lot of plants that use a gradient-like shading style to them like the peashooter and doom-shroom but they are actually light brush strokes. If you look closely at other spherical plants on Rich’s site you’ll notice that there tends to be a lot of strokes that start bright and get darker.

Zombies tend to have a more cell shading style for their skin. Most of the rough or hard-looking material tends to also be cell-shaded like the basic zombie, pogo zombie, and the zombie yeti while the softer, smoother, or leather-type clothing uses softer shading like the dancing zombie or catapult zombie.

Lighting and shading is also important. The lighting for all pieces of art comes from the top left. The shading also tends to be a little soft and doesn’t get very dark. The colors used are generally bright but unsaturated. There isn’t much color theory used besides a select few plants but using that theory makes your designs stand out.

Rich also has streamed in the past and has speedart videos on his YouTube channel, you can use these to get an understanding of how he does line art and shading as well.

This video has discussed the art of PvZ and its sequel in depth as well.

        One thing that you shouldn’t use as reference is the SVG art. Most of the high quality renders you see on the packaging or merchandise of PvZ uses SVG’s of the original art just so it can look better at any size, a standard practice among businesses. One way you can tell if it's an SVG is by the colors of the art.

Here's some examples of SVG versions on the top compared to the original art on the bottom. You’ll notice incorrect colors and less detail in general. You can view more in the galleries on the PvZ Fandom

How to make Lawn Backgrounds

When drawing a lawn background (LB), there are a couple things you must keep in mind.

  • Think of it as making a painting, there aren’t any black outlines or harsh colors used in any of the backgrounds unless you’re looking at the HD IPad backgrounds.
  • When adding other random assets to the LB, you will probably have to redraw them to make it fit within the style of the lawn, you should also stray away from putting them near the doors or the street because of the angle those are put at (refer to __Boxed#0469 background templates in the Drive)
  • When adding assets like a shovel or a flower to the LB, make sure there are no thick outlines as it will look completely out of place, don’t forget to give them shadows and depth too.

Editing XMLs

W.I.P

Most emitters use the following format for their timing

Starting, Time for Transition, Ending

Put the numbers in brackets for variation, use negatives for opposite movement

SpawnMinActive: minimum amount of particles allowed

SpawnMaxActive: maximum amount of particles allowed

SpawnRate: the amount of particles being spawned

SpawnMinLaunch: the minimum spawn rate allowed

SpawnMaxLaunch: the maximum spawn rate allowed

LaunchSpeed: how fast the particle will go

LaunchAngle: the angle of the launch

ParticleScale: how large the particle will be

ParticleDuration: how long the motion lasts

ParticleSpinSpeed: how much the particle will spin after spawning

ParticleStretch: how much the particle will stretch

ParticleLoops: if the particle loops or not

ParticleRed/Green/Blue: adds whatever color to the image (ranges from 0 to 1)

ParticleBrightness: controls the brightness of the particle

ParticleSpinAngle: the angle at which the particle spins

SystemDuration: how long the particle itself lasts

SystemAlpha: the opacity of the image.

SystemLoops: ?

CollisionReflect: how much bounce the particle will have

CollisionSpin: how much the particle will spin after hitting the ground

EmitterRadius: radius of which it is emitted

EmitterOffset: ?

Additive: ?

Image: what file will be used, eg: IMAGE_DIRTSMALL. use underscores instead of spaces

ImageFrames: amount of frames used

ImageRow: amount of rows used

<Field Types>

Acceleration: controls how fast the particle goes

Shake: how much the particle shakes

GroundConstraint: how long until the particle stops falling (if there is none it will fall through the ground)

Position: controls the positioning of the particle

Friction: the slipperiness of the particle

PopStudio Info

Considering that PopStudio has a lot to offer it might as well have its own section.

Package: This tab has all the stuff for unpacking and repacking the game with your custom assets. Unpack gets all the internal files out of the game and Pack puts it all of it back in. Unpacking Mode is usually only for ports of the game, most of them use the pak mode but the mobile ports use rsb mode. Convert all special images to png converts all the different file types you would find in the ports to pngs such as tex and ptx, these can also be deleted with the setting turned on below it.

Atlas: This tab deals with the cutting and building of atlases, which are the big images you would find in the majority of the mobile ports. You can either cut them which separates every image or build them which combines them all. Path of cut atlas is the destination of where the atlas is at. Path of out folder is where all the images will be put, usually you want to make a separate folder so the current one doesn’t get flooded with pngs. Path of cut info goes along with the Format of cut info, Format allows you to choose which format the resource file is in and it can help you find out what file you need to go into the path.

Image: This tab deals with the decoding and encoding of special images such as the ptx and tex image types you would find in the ports of the game. Decode decodes the image into a usable file type. Encode puts it back into the the format you want. The decoded special image path is the path wherever the image is at. The path of generated PNG image is the output location of the image, you have to name the image back to whatever the original name was and add the image format at the end too. Decoding Mode is what will decode or encode the image to.

Reanim: This tab has everything for decompiling and compiling reanim files. This tab is particularly special because you can turn your compiled reanims back into flas essentially, but with one caveat, the fla won't have any of its images and will lose all of its tweening. This means that you can take files from any version of the game and edit them. Path of in file is the input file, it can either be a .reanim or .compiled. Format of in file lets you choose what the format of the input file is, it has multiple settings to choose from such as the PC_Compiled, Phone64_Compiled, Raw_XML, ect. Path of out file outputs the file into whatever Format of out file is, this can be any of the settings in its drop down menu including Flash_Xfl_Folder which converts it into a editable fla file. When outputting the file, you have to name and add the format at the end of it (ex. Converting Zombie.reanim into a pc compiled you would have to name it “Zombie.reanim.compiled” with the settings in file - Raw_Xml out file - PC_Compiled) 

Particles: This tab has everything for decompiling and compiling XML files. The same stuff from the Reanim section applies here since the majority of it is the same. The only difference is that XML’s can’t be animated so you have to edit them through Notepad.
